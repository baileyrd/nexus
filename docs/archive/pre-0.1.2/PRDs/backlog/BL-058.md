# BL-058: Terminal URL chip extraction

**Status:** closed — 2026-05-06

**Source**: Terminal Integration Assessment (2026-05-06) — gap #2; CommandBook evaluation (2026-05-06)
**Files**: `shell/src/plugins/nexus/terminal/{urls.ts (new), urlExtractor.ts (new), UrlChips.tsx (new), TerminalView.tsx, index.ts}`
**Related**: `crates/nexus-terminal/src/urls.rs` (the Rust core, untouched)

The CommandBook-style URL pin strip is now mounted above the terminal output pane. Detection is byte-for-byte identical to the Rust core's `urls.rs` — the regex set was ported to TypeScript so the shell doesn't pay an IPC round-trip per line, and so the same input produces the same matches whether a tool consumes them server-side (AI / agent surfaces) or in the shell.

**Detection (`urls.ts`).** Three regex families: `https?://[^\s()\[\]<>"']+`, `file://[^\s()\[\]<>"']+`, and `\b(localhost|127\.0\.0\.1):\d+[^\s()\[\]<>"']*`. Trailing `.,;:!?)]}>` is stripped from the tail of every match (sentence-end punctuation). `localhost:PORT` and `127.0.0.1:PORT` matches are rewritten with a synthesised `http://` prefix so the OS opener accepts them. Overlapping matches are resolved in favour of the more specific scheme — `https://localhost:3000` surfaces as one `HttpHttps` entry, never also as `Localhost`.

**Stream extractor (`urlExtractor.ts`).** Wraps a UTF-8 `TextDecoder` in `stream: true` mode plus a small line buffer. Bytes flow in as `Uint8Array` chunks from the PTY; the extractor concatenates decoded text, splits on `\n` (CR is stripped before detection so a CRLF line-ending doesn't bleed into the URL), and emits matches once each line completes. `flush()` drains a buffered partial line at session end; `reset()` discards the buffer and the decoder's mid-multibyte state on session change. A multi-byte UTF-8 character split exactly across a chunk boundary is handled by the streaming decoder — covered by an explicit `café` test case.

**Chip strip (`UrlChips.tsx`).** Pill-style horizontal strip that renders above the xterm container. Most-recent-last, deduped by `resolved` URL, capped at five (`URL_CHIP_LIMIT`). Per-chip click routes through an `openExternal: (target) => Promise<void>` prop wired by `index.ts::activate` to `api.platform.shell.openExternal` — the chip component never imports `@tauri-apps/*` directly, keeping the WI-25 plugin-import-hygiene guardrail intact. Empty list collapses the strip so it doesn't claim vertical space when nothing's been detected.

**TerminalView wiring.** The existing `writeBytes` sink (the function the terminalStore calls with each output chunk) now also feeds the extractor; chips clear and the line buffer resets on session change. A small `pushUrl` helper handles dedupe + ring-buffer eviction.

**Tests.** 18 unit tests in `urls.test.ts` (re-exported via `tests/terminal-urls.test.ts`):

- detection: empty input, single https, trailing punctuation stripping, multiple matches in document order, bare localhost / 127.0.0.1, scheme-overlap resolution, file:// URLs, bare-localhost-without-port skip, in-word boundary skip.
- `resolveUrl`: pass-through and synthesised-prefix cases.
- extractor: per-line emission, partial-line buffering, `flush()` behaviour, `reset()` behaviour, CRLF stripping, split-multibyte-UTF-8 across chunks.

All 892 shell tests pass (was 874; +18 url).

**Deferred from the original DoD:**

- *FTP / SSH URL families* — the DoD prose mentioned them but the upstream `urls.rs` regex set covers HTTP(S) / file / localhost only. Port matched what's actually implemented; FTP / SSH would extend the upstream + shell symmetrically.
- *Per-chip dismiss* — only a strip-wide dismiss button is wired today. Adding per-chip dismiss would need a small per-instance hover affordance; `dismissUrls()` is sufficient for "I don't care about anything pinned right now".
