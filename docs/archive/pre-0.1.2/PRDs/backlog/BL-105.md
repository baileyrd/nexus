# BL-105: Theme builder — WCAG contrast checker

**Status:** closed — 2026-05-06

**Source**: BL-068 remaining feature (2026-05-06)
**Files**: `shell/src/plugins/nexus/themePicker/contrast.ts` (new), `ThemeBuilder.tsx`

New `contrast.ts` exports `parseHex`, `relativeLuminance`, and `contrastRatio(fg, bg)` using the WCAG 2.1 relative-luminance formula. `CONTRAST_PAIRS` in `ThemeBuilder.tsx` maps 12 text/syntax variable keys to their likely background pair (`--nx-text-*` → `--nx-bg-{primary,secondary}`, `--nx-syntax-*` and prose tokens → `--nx-editor-bg`). `VariableGroup` computes the effective background value (override ?? baseVars[bgKey]) for each paired row and passes it as `contrastBgValue` to `VarRow`. `VarRow` calls `contrastRatio`, and when both sides resolve to parseable hex a `ContrastBadge` renders between the text input and × button: green AAA (≥7:1), blue AA (≥4.5:1), or red ✗ Fail (<4.5:1), all with a tooltip showing the numeric ratio and pass/fail verdict. Badge updates live as the user types or uses the colour picker.
