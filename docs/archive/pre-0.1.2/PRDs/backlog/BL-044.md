# BL-044: MEM recall hotkey

**Status:** closed — 2026-04-29

**Delivered:** AI-MEMORY-LAYER-PLAN piece 3 lands as a default-off shell-side plugin at [`shell/src/plugins/nexus/recall/`](../../shell/src/plugins/nexus/recall/) that opens a Cmd+Shift+R (Ctrl+Shift+R on Linux/Windows) overlay, debounces 200ms-on-input, fans the trimmed query into `com.nexus.ai::semantic_search` (BL-040 handler 13) with `limit: 10`, scopes results to the BL-043 inbox path, and lets the user insert a quote-formatted snippet at the active editor caret (Enter) or copy it to the clipboard (Cmd/Ctrl+Enter). Reuses BL-032's overlay visual scaffold via duplication rather than a shared component — the overlays diverge enough (results list vs. streaming response panel; Up/Down navigation vs. context chips) that a content-adapter abstraction would have been more code than two ~210-line components. **Plugin** ([`recall/index.ts`](../../shell/src/plugins/nexus/recall/index.ts)): registers `nexus.recall.open` (keybinding `cmd+shift+r` / `ctrl+shift+r`) + `nexus.recall.close` (Esc, gated on the `nexus.recall.visible` context-key); contributes one `recall.hotkey` config key (default `Mod-Shift-r`, in CodeMirror parlance) for future user-customization; mounts `RecallOverlay` in the chrome `overlay` slot at priority 26 (one above the BL-043 capture overlay). Catalog-registered in `DEFAULT_OFF_PLUGINS` next to `linkSuggestPlugin` with the same comment rationale (depends on the AI plugin's embedding provider being configured; pairs naturally with `nexus.memory` so the inbox-scope filter has something to bind to). **Store** ([`recallStore.ts`](../../shell/src/plugins/nexus/recall/recallStore.ts)): zustand store mirroring the cmd-i shape — `{ visible, query, results, selectedIndex, status, error, currentRequestId }` with `open / close / setQuery / beginSearch / setResults / setError / moveSelection / setSelectedIndex` actions. `setResults(reqId, matches)` drops stale callbacks via the request-id guard so a slow `semantic_search` round-trip can't repopulate a closed overlay or overwrite a fresher search; `moveSelection(±1)` clamps to `[0, results.length-1]` so arrow-key navigation can't fall off either end; `setResults` also reclamps `selectedIndex` when the new list is shorter than the previous selection. `close()` retains query/results so reopening with the hotkey doesn't flicker between empty and prior state — the next `open()` wipes them anyway. **Runtime** ([`recallRuntime.ts`](../../shell/src/plugins/nexus/recall/recallRuntime.ts)): `searchDebounced(api, query)` collapses keystroke bursts into a single `runSearch` call after `RECALL_DEBOUNCE_MS = 200` of quiet (the trailing edge wins; same value the BL-039 link-suggest poll settled on); empty/whitespace queries short-circuit to a clean state without firing a kernel call (no spinner-then-empty flicker). `runSearch` mints a `recall-<uuid>` request id, calls `semantic_search` with `{ query, limit: 10 }`, decodes the `{ matches: ChunkMatch[] }` response defensively (drops malformed entries lacking `file_path`, same shape as the BL-040 palette plugin's decoder), and applies `filterToInboxScope`. `insertSelectedSnippet()` reaches the active CodeMirror view via `editor/runtime.getActiveCmView()` (a sibling-plugin import — same pattern `editor/cm/linkSuggest.ts` already uses for BL-039) and dispatches a `{ from: head, to: head, insert: snippet }` change with the post-splice caret placed AFTER the inserted snippet so the user can keep typing without manually escaping the quote block; pulls focus back to the editor on success. `copySelectedSnippet()` writes through `navigator.clipboard.writeText` and degrades silently when the platform clipboard is unavailable. **Inbox-scope filter** ([`filterToInboxScope`](../../shell/src/plugins/nexus/recall/recallRuntime.ts)): v1 reads `memory.inboxPath` via `api.configuration.getValue` (default `Inbox.md` matching BL-043) and keeps only matches whose `file_path === inboxPath`. **Known v1 limitation**: when the filter would zero out a non-empty result list, the runtime degrades to surfacing the unfiltered set — recall with no results is worse UX than recall with out-of-scope results when the user explicitly typed a query that hit the index. The richer "from project" / multi-path / tag-based filter is BL-046 (code-aware capture) territory; the BACKLOG entry for BL-046 already calls this out. **Insert formatter** ([`insertFormat.ts`](../../shell/src/plugins/nexus/recall/insertFormat.ts)): pure function `formatRecallSnippet(match)` returns ``> {chunk_text trimmed, line-prefixed}\n>\n> — [[basename]]\n``. Strips a single trailing `.md` (case-insensitive) so the produced wiki-link resolves through Nexus's existing link resolver; preserves internal newlines by prefixing every line with `> ` so the snippet stays inside one quote block; trims surrounding whitespace; handles empty `chunk_text` without producing a malformed block (single `>` line + attribution). **Overlay** ([`RecallOverlay.tsx`](../../shell/src/plugins/nexus/recall/RecallOverlay.tsx)): autofocused `<input>` (RAF-deferred to dodge the mount race, same trick the command palette uses) + `ResultList` (renders `basename / chunk-excerpt` rows with `aria-selected` on the highlighted entry; clicking a row sets `selectedIndex` so mouse and keyboard navigate the same selection state). The `onKeyDown` handler routes Esc → `cancelPendingSearch + close`, Arrow keys → `moveSelection(±1)`, Enter → `insertSelectedSnippet + close`, Cmd/Ctrl+Enter → `copySelectedSnippet + close` (close runs in `.finally` so a clipboard failure still dismisses the overlay). Backdrop click closes; backdrop visual matches the Cmd+I overlay (`oklch(0 0 0 / 0.35)` over the workspace, centred 640px-wide dialog at `paddingTop: 120`). The overlay also runs `cancelPendingSearch` from a `useEffect` cleanup when `visible` flips false, so a late timer can't repopulate a freshly-closed overlay even if the close was via backdrop click rather than Esc. **Coverage**: `pnpm --filter nexus-shell test` 521 passing + 1 skipped (was 499 / 1; +22 net): 8 store tests in [`recallStore.test.ts`](../../shell/src/plugins/nexus/recall/recallStore.test.ts) covering `open()` reset, `close()` retention, `setQuery` non-touch of results, request-id guard on `setResults`, arrow-key clamp at both ends, no-results clamp, reclamp on shorter results list, and `setError` state shape; 5 formatter tests in [`insertFormat.test.ts`](../../shell/src/plugins/nexus/recall/insertFormat.test.ts) covering single-line snapshot, multi-line line-prefixing, whitespace trim, empty-chunk handling, and the `.md`-only suffix-stripping rule (`.markdown` and no-suffix names pass through unchanged); 9 runtime tests in [`recallRuntime.test.ts`](../../shell/src/plugins/nexus/recall/recallRuntime.test.ts) covering `filterToInboxScope` keep / passthrough / degrade-to-unfiltered, `runSearch` decoder mapping (incl. malformed-entry rejection), inbox-filter wire-up, kernel-error → status:error, debounce empty-query short-circuit, debounce burst-collapse (3 rapid calls → 1 kernel hit with the last query), and a pin-test on `RECALL_DEBOUNCE_MS = 200`.
