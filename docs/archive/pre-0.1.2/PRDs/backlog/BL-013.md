# BL-013: Terminal event subscription over plugin IPC

**Status:** closed — 2026-04-28

**Delivered:** PRD-09 §11.2's `subscribe_events` story now reaches the kernel bus. `TerminalCorePlugin::with_event_bus` (the same builder bootstrap already calls in `crates/nexus-bootstrap/src/lib.rs:875`) now also subscribes to the in-memory server's `TerminalEvent` mpsc and spawns a `nexus-terminal-lifecycle` forwarder thread that republishes every event onto the kernel bus as `com.nexus.terminal.events.<session_id>` (new `EVENT_LIFECYCLE_PREFIX` constant, exported from `nexus_terminal`). The payload is the `TerminalEvent` itself — `serde(tag = "kind")` keeps `session_created` / `output_received` / `pattern_matched` / `session_closed` distinguishable, so a single `EventFilter::CustomPrefix("com.nexus.terminal.events.")` subscription gives plugins everything they need to react to lifecycle and per-line output without re-implementing pump loops; an `EventFilter::CustomExact` subscription against a specific `<session_id>` topic is also viable for remote-terminal style consumers. The byte-stream topic added in WI-12 (`com.nexus.terminal.output.<session_id>`) stays the right channel for xterm-style frontends; the lifecycle topic complements it. The forwarder owns the receiver end only — it never holds the server lock — so it adds zero contention to dispatch. Drop ordering is correct: `LifecycleForwarderHandle::drop` flips a stop flag and joins (worst-case ~one `LIFECYCLE_RECV_TIMEOUT_MS = 100ms` tick); the thread also exits naturally when the server's senders go away, so a forwarder outliving its server still terminates. New `TerminalEvent::session_id()` helper extracts the id from any variant for topic-suffix derivation. Five new unit tests (`crates/nexus-terminal/src/core_plugin.rs`) verify each variant lands on the bus end-to-end (`SessionCreated`, `OutputReceived`, `PatternMatched`, `SessionClosed`, plus a topic-suffix invariant). Coverage: `cargo test -p nexus-terminal`: 253 passing (+5 new); `cargo clippy -p nexus-terminal --all-targets -- -D warnings`: clean; `cargo check --workspace`: clean. Out of scope per the BL-013 backlog note: a generic plugin-host stream convention — the `Custom` event with a per-session prefix already gave us the ergonomics the backlog called out, and codifying a wider streaming trait is a separate cross-cutting effort the backlog can pick up when a second streaming consumer (file watcher, MCP notifications) appears.
