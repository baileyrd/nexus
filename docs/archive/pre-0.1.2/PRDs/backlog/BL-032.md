# BL-032: Cmd+I command-anywhere AI overlay

**Status:** closed — 2026-04-28

**Source**: [../AI-AMBIENT-COPILOT-PLAN.md](../AI-AMBIENT-COPILOT-PLAN.md) pattern 1.
**Effort**: Medium. **Crate / Package**: `shell/src/plugins/nexus/ai/`.
**Delivered:** Phase-1 foundation that the BL-010 / BL-011 / BL-033 / BL-034 / BL-044 surfaces all hang off — a modal overlay invocable from any focused surface (`Ctrl+I` / `Cmd+I`) that takes a free-form prompt, gathers per-surface context via a registry of contributor adapters, and routes the assembled prompt through `com.nexus.ai::stream_chat` with an isolated session id so streaming events don't bleed into the chat panel store. Shipped commit `715d82d` (PR #27).

- **Context-contributor registry** ([`contextContributors.ts`](../../shell/src/plugins/nexus/ai/contextContributors.ts)): singleton-module registry mirroring `fencedCodeRegistry` (BL-008). Each contributor returns a `ContextContribution { surfaceId, chips: ContextChip[], promptBlock?: string, chipPromptBlocks?: Record<chipId, string> }` from an async `collect()` call. `ContextChip` carries `{ id, label, kind }` with `kind ∈ 'file'|'selection'|'block'|'row'|'node'|'query'|'note'` so chips render with kind-prefix glyphs without surface-specific renderers. `assemblePrompt(contributions, removedChipIds)` composes the final user-prompt prefix in two modes — per-chip (when `chipPromptBlocks` is set, only kept chips' fragments emit, in chip order) and coarse fallback (drop the surface's whole `promptBlock` only when ALL of its chips are removed). Adapter contract is async + null-returning so a no-op contribution (e.g. editor with no tab focused) just falls out of the assembled prompt.
- **Editor adapter** ([`editorContextAdapter.ts`](../../shell/src/plugins/nexus/ai/editorContextAdapter.ts)): first surface adapter, ships in this PR. Reads `useEditorStore` for the active tab + body snapshot and `getActiveCmView()` (sibling-plugin import — same pattern `editor/cm/linkSuggest.ts` uses for BL-039) for the live CodeMirror selection. Emits two chips per activation when there's a selection (file + selection) and one when there isn't (file only); body cap is `8 KB` per file with a line-boundary truncate so the model never sees a half-fenced code block. Lives in the AI plugin's directory rather than the editor plugin's because the editor plugin is default-on and we don't want it depending on a default-off plugin.
- **Store** ([`cmdIStore.ts`](../../shell/src/plugins/nexus/ai/cmdIStore.ts)): zustand `useCmdIStore` with `{ visible, prompt, chips, removedChipIds, status: 'idle'|'collecting'|'submitting'|'streaming'|'done'|'error', responseText, error, currentRequestId }`. `open()` clears everything per activation so chips/response from a prior summon don't bleed in; status starts `'collecting'` until the runtime hydrates the chips. Tail-chunk guards on `appendResponseChunk` / `finishResponse` reject mismatched `currentRequestId` so a late `stream_chunk` can't append into a closed overlay.
- **Runtime** ([`cmdIRuntime.ts`](../../shell/src/plugins/nexus/ai/cmdIRuntime.ts)): `openCmdI()` resolves every registered contributor (pure — does NOT call the kernel), pushes chips into the store, and flips `visible: true`. `submitCmdI()` mints a `cmdi-<uuid>` session id (segregates events from the persistent chat panel), assembles the prompt via `assemblePrompt(contributions, new Set(removedChipIds))`, and fires `com.nexus.ai::stream_chat` with `tools: 'auto'` to advertise the BL-016 registry. `routeStreamEvent` is the unified handler the AI plugin's single `com.nexus.ai.stream_*` subscription installs on activate; it dispatches each chunk/done event to either the chat store or the cmd-I store based on the session-id correlation, so the runtime doesn't open a second subscription.
- **Overlay UI** ([`CmdIOverlay.tsx`](../../shell/src/plugins/nexus/ai/CmdIOverlay.tsx)): chrome `overlay`-slot modal, centred 640px-wide dialog with `oklch(0 0 0 / 0.35)` backdrop. Autofocused `<textarea>` (RAF-deferred to dodge the mount race), `ContextChipRail` rendering the chips read-only in v1 (click-to-remove + the model switcher land in BL-033), streaming response panel below the input, `Esc` closes, `Mod+Enter` submits, plain `Enter` inserts a newline (so multi-line prompts work without bracketed-paste tricks). Backdrop click closes.
- **Plugin wiring** ([`index.ts`](../../shell/src/plugins/nexus/ai/index.ts)): manifest contributes commands `nexus.ai.cmdI.open` (keybinding `ctrl+i` / `cmd+i`) and `nexus.ai.cmdI.close` (Esc, gated on the `nexus.ai.cmdI.visible` context-key); the overlay mounts in the `overlay` slot via `views.registerPanel(VIEW_ID_CMD_I_OVERLAY, …)`. `setCmdIApi(api)` hands the kernel handle to the runtime so submit can call `api.kernel.invoke(...)` without a DI dance. `useCmdIStore.subscribe(...)` flips the context-key when `visible` changes so the Esc keybinding only competes for the chord while the overlay is up.
