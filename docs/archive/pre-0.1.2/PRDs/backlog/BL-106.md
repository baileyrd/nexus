# BL-106: Theme builder — light/dark side-by-side preview

**Status:** closed — 2026-05-06

**Source**: BL-068 remaining feature (2026-05-06)
**Files**: `shell/src/plugins/nexus/themePicker/ThemeBuilder.tsx`, `themePickerStore.ts`, `ThemePicker.tsx`, `contrast.ts`

"Dual mode" checkbox added to the Build tab. When enabled the modal expands to 960 px and the variable editor splits into two scrollable columns (☀ Light | 🌙 Dark) each with independent override sets (`builderLightOverrides` / `builderDarkOverrides` in the store). A column header click makes that column "live" (immediately applies its overrides via `set_plugin_overrides`) and shows a ● live indicator. Hue-lock checkbox: when on, editing a hex colour in one column propagates the hue + saturation delta to the same key in the other column (lightness preserved) using `applyHueSatDelta` from `contrast.ts`, with the pre-edit value as the delta reference. TOML export in dual mode generates two separate theme manifests (light + dark) in the textarea, each with `supports = ["light"]` / `supports = ["dark"]`. "Apply now" button label changes to "Apply light" / "Apply dark" to clarify which column is being sent. "Reset all" clears both override sets. Override count in the footer totals across both columns.
