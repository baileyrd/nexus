# BL-026: MCP Resource Enumeration

**Status:** closed — 2026-04-28

**Delivered:** PRD-14 §7 lands as a focused server-side addition to [`crates/nexus-mcp/src/server.rs`](../../crates/nexus-mcp/src/server.rs). `NexusMcpServer`'s `ServerHandler` impl now overrides `list_resources` and `read_resource` in addition to the existing `get_info` / `call_tool` / `list_tools`; the capability advertisement in `get_info` gains `.enable_resources()` so MCP clients negotiate the resource surface during initialize. Notes are exposed under the URI scheme `mcp://nexus/notes/<vault-relative-path>` (the listing root `mcp://nexus/notes` is non-readable). `list_resources` reuses `com.nexus.storage::query_files` (handler id 1) — same IPC path the existing `nexus_list_notes` tool uses — and maps each `{ path, size_bytes }` record to a `RawResource` with `mime_type = "text/markdown"`, description `"Markdown note in the Nexus forge"`, name set to the file basename, and size clamped to `u32::MAX` via `u32::try_from(...).unwrap_or(u32::MAX)` (no silent `as u32` truncation). `read_resource` dispatches `parse_note_uri(&request.uri)` first — non-`mcp://nexus/notes/` URIs and the bare notes-root URI return `ErrorData::resource_not_found` — then routes the relative path through `com.nexus.storage::read_file` (id 2), decodes the byte array via `String::from_utf8_lossy`, and returns a single `ResourceContents::TextResourceContents { uri, mime_type: "text/markdown", text }`. No new external dependencies; no IPC contract drift; no `nexus-storage` linkage (the existing `storage_call` helper handles the IPC dispatch). The host plugin (`com.nexus.mcp.host`) and `com.nexus.storage` handlers are untouched — BL-026 is server-side only. `mcp://nexus/db/...` is intentionally deferred per the PRD-14 §7.3 note: that shape needs a `com.nexus.bases::list_columns`-equivalent handler that doesn't exist yet. Coverage: 3 new unit tests at the bottom of `server.rs` — `parse_note_uri_extracts_path` (valid path, missing prefix, notes-root URI), `build_note_resource_sets_uri_mime_and_size`, `build_note_resource_clamps_oversize_to_u32_max`. Total `cargo test -p nexus-mcp`: 21 passing (was 18).
