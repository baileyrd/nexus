**Delivered:** AI-AMBIENT-COPILOT-PLAN pattern 5 + the Notion-block follow-up "block AI actions via `com.nexus.ai`" land as a shared in-tree registry plus four shipped built-ins. **Extension-API contract** ([`packages/nexus-extension-api/src/index.ts`](../../packages/nexus-extension-api/src/index.ts)): new `AiActionSurface = 'editor.selection' | 'block' | 'canvas.node'`, three discriminated payload interfaces (`AiActionEditorSelectionContext` carries `relpath` + `selection` + `selectionRange`; `AiActionBlockContext` carries `relpath` + `blockId` + `blockText`; `AiActionCanvasNodeContext` reserved for BL-038), `AiAction { id, label, surfaces, run(ctx) }`, and `AiAPI { registerAction(action): Disposable }`. `NexusPluginContext` gains an `ai: AiAPI` field — types-only at the package boundary; runtime impl is supplied by the host (mirrors how `editor` / `ui` / `workspace` already split). **Singleton registry** ([`shell/src/plugins/nexus/ai/actions/registry.ts`](../../shell/src/plugins/nexus/ai/actions/registry.ts)) mirrors the `contextContributors.ts` shape: `register(action)` returns an idempotent disposer, `list()` snapshots in registration order, `actionsForSurface(surface)` filters by surface tag, `invoke(action, ctx)` catches + `console.warn`s thrown handlers so a misbehaving action can never crash the menu, `_resetForTests()` for isolation. **Built-ins** ([`shell/src/plugins/nexus/ai/actions/builtins.ts`](../../shell/src/plugins/nexus/ai/actions/builtins.ts)): `nexus.ai.summarize`, `nexus.ai.rewrite`, `nexus.ai.translate`, `nexus.ai.explain`, every one wired to `surfaces: ['editor.selection', 'block']`. Each `run(ctx)` extracts the selection/block text, builds a per-action system prompt + user turn, and dispatches through a new [`aiRuntime.streamChat({ messages, system, tools: 'auto' })`](../../shell/src/plugins/nexus/ai/aiRuntime.ts) helper that wraps `com.nexus.ai::stream_chat` (handler id 6) so the BL-016 tool registry is advertised. Translate honours an optional `targetLanguage` (default `English`); the action's `label` reflects the active language (`Translate to French`). **Output routing**: v1 surfaces results via `api.notifications.show` (clipped to 240 chars in the toast) plus a `console.info` mirror for the developer console, because no `com.nexus.editor::insert_after_selection` IPC exists yet and the task explicitly forbids inventing one — when that IPC lands, swap `surfaceResult` to splice inline. **Plugin registration**: [`shell/src/plugins/nexus/ai/index.ts`](../../shell/src/plugins/nexus/ai/index.ts) `activate()` calls `registerBuiltinAiActions(api)` alongside the existing `registerEditorContextAdapter()`; same disposer-not-tracked rationale (module-scope singleton + no `deactivate` hook + registry tolerates hot-reload duplicates). **Deferred**: hooking the registry into the editor's right-click selection context menu and the block-handle menu is parked for a follow-up — the existing block-handle UI in [`shell/src/plugins/nexus/editor/cm/blockHandle.ts`](../../shell/src/plugins/nexus/editor/cm/blockHandle.ts) renders rows imperatively from a hard-coded `items` array, and the editor selection context menu doesn't exist as a wired surface today (right-click selection just gets the browser default).
