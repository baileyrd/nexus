# BL-041: AI-DIR background indexing daemon

**Status:** closed — 2026-04-29

**Delivered:** AI-INTEGRATION-DIRECTIONS direction 8 lands as a new [`crates/nexus-ai/src/indexing_daemon.rs`](../../crates/nexus-ai/src/indexing_daemon.rs) module that subscribes to the existing storage file-watcher, debounces bursts of saves, and re-runs `rag::index_file` (the BL-019 throughput-optimised path that bypasses the cache for >1k chunks per batch) without the user touching `nexus ai index_file`. **Daemon** ([`indexing_daemon.rs`](../../crates/nexus-ai/src/indexing_daemon.rs)): `IndexingDaemon::start(ctx, status, embedder_factory)` spawns a dedicated `nexus-ai-indexing-daemon` thread that owns its own current-thread tokio runtime — sidesteps any assumption about the frontend's runtime topology and keeps `ctx.ipc_call` (async) reachable without blocking the kernel scheduler. Pure-logic `Debouncer { touched: HashSet, deleted: HashSet, last_event: Option<Instant>, debounce: Duration, max_batch: usize }` collapses rapid saves into one re-index per path per flush window (default `DEFAULT_DEBOUNCE = 2 s` / `DEFAULT_MAX_BATCH = 32`); time is injected so unit tests are deterministic. Delete supersedes touch (`a.md` deleted then touched cancels the delete; touched then deleted fires a delete). `event_to_msg` translates `com.nexus.storage.file_{created,modified,deleted,renamed}` custom events into `DaemonMsg::{Touched, Deleted}` and filters non-markdown extensions defensively (the storage watcher already does this; double-check guards future relaxations). **Worker loop**: 200 ms tick that drains both bus events (via `EventFilter::CustomPrefix("com.nexus.storage.file_")`) and a direct `mpsc::UnboundedReceiver<DaemonMsg>` (used by tests / future `index_trigger` handler), updates `pending_files` under one brief write lock, then calls `Debouncer::maybe_flush(Instant::now())`. On flush: deletions route through `com.nexus.storage::vector_delete_by_file` (handler 19); touches fetch blocks via `com.nexus.storage::query_blocks` (handler 21), `decode_blocks` re-shapes the JSON return into the `(u64, String, String, Option<i32>)` `BlockTuple` shape `rag::index_file` consumes (accepts both the raw array and `{ blocks: [...] }` wrapper for forward-compat), then `rag::index_file(ctx, embedder.as_ref(), &path, &blocks)` runs. Per-IPC timeout `IPC_TIMEOUT = 30 s`. Errors are captured into `IndexStatus.last_error` but never abort the daemon — the next batch retries; `last_error` is cleared on the next successful flush. **Status state**: `IndexStatus { indexed_files: u64, pending_files: u64, total_seen: u64, last_error: Option<String>, running: bool }` lives behind `SharedStatus = Arc<RwLock<IndexStatus>>`; cheap to clone, briefly write-locked on each event arrival and the snapshot read. **IPC handler** ([`crates/nexus-ai/src/core_plugin.rs`](../../crates/nexus-ai/src/core_plugin.rs)): new `HANDLER_INDEX_STATUS: u32 = 14` on `com.nexus.ai`, args `{}`, returns `IndexStatus` verbatim. Dispatches in both sync and async paths so the shell's standard `kernel_invoke` async route works alongside CLI sync probes. **Plugin lifecycle**: `AiCorePlugin` gains `index_status: SharedStatus` (allocated unconditionally in `new()` so a fast `index_status` read returns a meaningful "never ran" view) and `indexing_daemon: Option<IndexingDaemon>` (spawned in `wire_context`, *not* `on_start`, because `on_start` runs before context wiring and the daemon needs `ctx.subscribe` + `ctx.ipc_call`). The embedder is supplied through an `EmbedderFactory = Arc<dyn Fn() -> Option<Box<dyn EmbeddingProvider>> + Send + Sync + 'static>` closure that captures the live `embed_config` handle so the daemon picks up `set_config` updates without a restart. `on_stop` signals shutdown via `Arc<AtomicBool>` and joins the worker. **Manifest** ([`crates/nexus-bootstrap/src/lib.rs`](../../crates/nexus-bootstrap/src/lib.rs)): `("index_status", HANDLER_INDEX_STATUS)` appended to the `com.nexus.ai` core manifest; `LifecycleFlags { on_init: true, on_stop: true, ..NONE }` so the kernel calls `on_stop` on graceful shutdown. **Shell badge** ([`shell/src/plugins/nexus/statusBar/IndexingStatus.tsx`](../../shell/src/plugins/nexus/statusBar/IndexingStatus.tsx)): polls `com.nexus.ai::index_status` every 2 s, renders a compact `dot + label` (green = idle, blue + glow = pending, red = error) with the indexed-files count or pending count or last-error message; latches the last-good snapshot so a transient kernel hiccup doesn't blank the badge; renders `null` when the daemon has never run (no AI / no embedder configured) so users don't see a permanently dead grey dot.
