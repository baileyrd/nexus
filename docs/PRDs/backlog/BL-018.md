# BL-018: AI Token Budget Enforcement

**Status:** closed — 2026-04-28

**Delivered:** PRD-12 §12 lands as a small library-only crate addition. New [`crates/nexus-ai/src/tokens.rs`](../../crates/nexus-ai/src/tokens.rs) defines a `TokenCounter` trait + `ApproxTokenCounter` that returns `(text.len() + 3) / 4` — the 4-chars-per-token approximation Anthropic's docs recommend for English — so no `tiktoken` / `tokenizers` dependency is pulled in. `TokenBudget::new(total, reserved)` carves out the response reservation up-front; `allocate(kind, tokens)` charges against a `ContextSourceKind` (`RagChunk` / `SystemPrompt` / `UserPrompt` / `Other`), rejecting and leaving state unchanged on overflow; `remaining` / `used` / `utilization` round out the accounting surface. `BudgetWarning::NearLimit { utilization }` fires when realised utilisation crosses the 80 % threshold; `BudgetWarning::SourceDropped { kind, tokens }` records every chunk that didn't fit. RAG-side: `rag::build_rag_prompt_budgeted(sources, &mut budget, &counter) -> (String, Vec<BudgetWarning>)` sorts sources by descending score, charges each rendered chunk under the upper-bound `Source 99: [[path]]\n<text>\n\n` template, drops the lowest-scoring chunks first when they don't fit, and surfaces the warnings. The pre-existing `build_rag_prompt` stays `pub` as a thin wrapper around it (constructs an effectively unlimited budget, discards warnings) so every existing caller (`rag::query`, `stream_ask`) keeps its byte-for-byte legacy behaviour. `AiConfig` gained `context_window: u32` (default 8192) and `reserved_response_tokens: u32` (default 1024); the existing `max_tokens` (provider request body cap on generation length) is untouched. Coverage: 10 new unit tests — 6 in `tokens.rs` (`approx_counter_returns_chars_div_4_rounded_up`, `budget_remaining_subtracts_reserved_and_used`, `budget_allocate_returns_false_when_overflow`, `budget_utilization_reports_used_over_available`, plus edge cases for zero-available utilisation and additive same-kind allocation) + 4 in `rag.rs` (`budgeted_prompt_includes_all_sources_when_under_budget`, `budgeted_prompt_drops_lowest_score_first_when_over_budget`, `budgeted_prompt_emits_near_limit_warning_at_80pct`, `budgeted_prompt_with_zero_sources_returns_legacy_default`). Out of scope per the BL-018 plan: wiring the budget into `stream_chat` / `stream_ask` provider request paths — that ships when a downstream caller needs it.
