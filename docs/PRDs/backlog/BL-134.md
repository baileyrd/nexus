## BL-134 Phase 2b-ii ‚Äî AiEvent republisher

Typed AiEvent republisher for `com.nexus.ai.stream_chunk` + `com.nexus.agent.round_proposed`. Protocol change: `SessionRunArgs` gained an optional `session_id: Option<String>` field ‚Äî agent uses the caller-supplied id when present, self-allocates otherwise (preserving legacy non-runtime callers). Runtime worker pre-allocates a UUID at submit time, injects it into the session args before forwarding to `session_run`, and registers the `session_id ‚Üí task_id` correlation in `Store::session_to_task`; cleanup runs on terminal event + cancel-before-start. New `nexus_ai_runtime::republisher` module ships two pure helpers (`extract_session_id`, `translate_bus_event`) under unit tests. `wire_context` spawns a `republish_loop` on the worker pool that subscribes to both topics via `EventFilter::CustomExact`, looks up the correlation, and republishes as typed `AiEvent::{TokenChunk, RoundProposed}` under `com.nexus.ai.runtime.*` ‚Äî events from sessions outside the runtime's correlation map are silently dropped. End-to-end test against `build_cli_runtime` exercises the full submit‚Üípublish‚Üítranslate‚Üírepublish path. 47/47 in-crate tests; 8/8 ai_runtime_ipc integration tests including the new e2e.

## BL-134 Phase 4 ‚Äî indexing-daemon migration

Indexing-daemon migration via Pattern 1 (encapsulated pool-handle injection). New `nexus_ai_runtime::shared_pool_handle()` reads from a module-private `OnceLock<Handle>` filled in the runtime's `wire_context` after `WorkerPool::start`; bootstrap reorders so `ai-runtime` registers BEFORE `ai`. `nexus-ai::indexing_daemon::start` now prefers the shared handle (`handle.block_on(worker_loop(...))`) and only falls back to a bespoke `tokio::runtime::Builder::new_current_thread()` build when the runtime plugin isn't wired (e.g. unit-test bootstrap), logging the fallback at warn. New bootstrap integration test (`bootstrap_publishes_shared_pool_handle_for_indexing_daemon`) plus runtime-side `publish_shared_handle` unit test pin the wiring contract. `nexus-ai` gains a direct dep on `nexus-ai-runtime` ‚Äî allowed by `dep_invariants` since the rule only forbids frontend crates (CLI/TUI/MCP) from depending on the runtime, not sibling service plugins.
