# BL-031: Bases cell / row clipboard

**Status:** closed — 2026-04-28

**Delivered:** Mod-C / Mod-X / Mod-V on a focused `.bases` leaf now copy a rectangular cell range (anchor + shift-click), a single selected row, or paste either of those — gated on `when: 'bases.focused && !bases.editing'` so the chords don't shadow `CellEditor` input. Each paste enrols **one** `HistoryEntry` so undo/redo round-trips the bulk operation, satisfying the BL-030 pairing requirement. New [`shell/src/plugins/nexus/bases/clipboard.ts`](../../shell/src/plugins/nexus/bases/clipboard.ts) owns the pure helpers (TSV serialization, JSON payload format, parser, per-cell coercion via `fieldTypes.ts`); new [`tableClipboard.ts`](../../shell/src/plugins/nexus/bases/tableClipboard.ts) is a registry-of-one (mirror of `activeBases.ts`) so the plugin's `cut/copy/paste` commands route through whichever table currently owns focus. Two typed clipboard MIME payloads ride alongside `text/plain` TSV: `application/x-nexus-bases-cells` for rectangular ranges and `application/x-nexus-bases-rows` for whole records — external apps still see the TSV grid, while cross-base paste preserves typing. Three paste modes land per the BL-031 plan: **range** (rectangle → tile/clip when source != range size; single-cell selection treats source as anchor + extent), **single-row** (record selected → shallow-merge by field name), and **no-selection** (typed `rows` payload or TSV-with-header → append new records via `base_record_create`). Cross-base coercion keys on `coerceValue(destKind, value, sourceKind)` — same-type values pass through unflagged, otherwise the per-cell coercion attempt either succeeds (numbers strip currency / percent symbols, text joins multi-select arrays with `, `, etc.) or drops to `null` and trips `coercedCount`. When `coercedCount > 0` the table emits a `bases:paste-coerced` notification via `api.notifications.show`. Read-only column types (`formula` / `rollup` / `lookup` / `uuid` / `relation`) are skipped at both copy and paste edges via `isPasteable`. Cell-range selection lives on a new `cellSelection: CellRange | null` field in `BasesTabState` — disjoint from the existing `selectedRecordId` (setting one clears the other) so the two selection modes don't shadow each other in the keyboard handlers. New `bases.editing` context-key is published from `BasesTable` on every `editing` state change so the `when:` clause resolves correctly. Coverage: `pnpm --filter nexus-shell test` 444 passing including 14 new BL-031 tests in [`basesStore.test.ts`](../../shell/src/plugins/nexus/bases/basesStore.test.ts) covering `normalizeRange` flip, `rangeContains` edges, `cellToTsv` / `cellsToTsv` / `rowsToTsv` / `parseTsv` round-trips, six `coerceValue` scenarios, `isPasteable`, and the disjoint-selection-modes invariant.
