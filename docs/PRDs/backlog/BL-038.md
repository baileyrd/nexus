# BL-038: AMB citations

**Status:** closed — 2026-04-29

**Delivered:** AI-AMBIENT-COPILOT-PLAN pattern 8 lands as a numbered, line-aware citation contract that flows from `rag::query` all the way through the streaming `stream_ask` IPC handler into the chat view, where `[N]` markers in the model's answer become clickable superscript chips that open the source file at the cited line. **Backend** ([`crates/nexus-ai/src/rag.rs`](../../crates/nexus-ai/src/rag.rs)): new `Citation { index: u32, file_path: String, block_id: u64, start_line: Option<u32>, end_line: Option<u32>, excerpt: String, score: f32 }` derives `Serialize` / `Deserialize` / `TS` / `JsonSchema` (the latter two gated on the existing `ts-export` feature). `RagResponse` gains a `citations: Vec<Citation>` field parallel to the legacy `sources: Vec<ChunkMatch>` — `sources` stays in place for backwards compat per the BL-038 task allowance ("ship side-by-side this round"). The pure helper `build_citations(ctx, sources, answer)` does three things in order: (1) groups source chunks by `file_path` and fans out one `com.nexus.storage::query_blocks` (handler 21) IPC call per unique path, indexes the returned `BlockRecord`s by `id`, and joins `start_line` / `end_line` onto each citation; (2) numbers citations 1-based in source order (which is descending by score, since that's what `vectorstore::search` returns); (3) when the model emitted parseable `[N]` markers in the answer, renumbers citations by first-occurrence so the chip ordering matches reading order — a tiny inline byte-walker covers `\[(\d+)\]` without pulling a regex dep, skips out-of-range markers, and preserves trailing source-order slots for citations the model didn't reference. Enrichment failures (storage offline, file moved, index lag) degrade to `start_line: None` rather than failing the whole query, satisfying the file-as-truth invariant: the answer + chunk text remain valid even if the index drifts. Excerpts are clipped to `CITATION_EXCERPT_MAX_CHARS = 200` via a char-boundary-safe `truncate_excerpt` (won't split a UTF-8 codepoint). **Prompt teach-cite** ([`build_rag_prompt`](../../crates/nexus-ai/src/rag.rs)): `RAG_PROMPT_HEADER` now instructs the model to "Cite sources as [N] where N is the 1-based position in the SOURCES list below (for example, [1] or [2])" alongside the legacy `[[file_path]]` form so models that prefer one or the other both produce parseable output. The numbered `Source N:` body lines are unchanged so the same prompt still works for older callers. **Streaming integration** ([`crates/nexus-ai/src/core_plugin.rs`](../../crates/nexus-ai/src/core_plugin.rs)): `handle_stream_ask` (HANDLER_STREAM_ASK = 7) now calls `rag::build_citations(&ctx, &sources, &text)` after the chat stream completes, then surfaces `citations` on both the `com.nexus.ai.stream_done` event payload and the IPC return value so the shell receives them through whichever path resolves first (the event handler typically wins; the invoke-result fallback covers dropped events). `handle_ask` (HANDLER_ASK = 1) inherits citations transparently because it returns the full `RagResponse`. **ChunkMatch ts-export gate** ([`crates/nexus-ai/src/vectorstore.rs`](../../crates/nexus-ai/src/vectorstore.rs)): `ChunkMatch` now derives `TS` / `JsonSchema` under `ts-export` so `RagResponse.sources: Vec<ChunkMatch>` compiles when the drift-check feature flag is on. **TS bindings** ([`packages/nexus-extension-api/src/generated/ipc/`](../../packages/nexus-extension-api/src/generated/ipc/)): three new generated files committed by the drift script — `Citation.ts`, `RagResponse.ts`, `ChunkMatch.ts`. `bash scripts/check_ipc_drift.sh` is clean. **Shell types** ([`shell/src/plugins/nexus/ai/aiStore.ts`](../../shell/src/plugins/nexus/ai/aiStore.ts)): new `AiCitation { index, path, blockId, startLine: number|null, endLine: number|null, excerpt, score }` mirrors the kernel struct (snake_case to camelCase). The assistant turn variant gains `citations: AiCitation[]` (initialized to `[]` in `startAsk`); `finishStream` accepts an optional `citations` argument paralleling `sources`; `decodeTurn` rehydrates citations from persisted sessions, defensively dropping malformed entries. **Runtime coercion** ([`aiRuntime.ts`](../../shell/src/plugins/nexus/ai/aiRuntime.ts)): new `coerceCitations(raw)` mirrors the existing `coerceSources` shape, sorts by `index` for stable rendering even if the kernel order shifts, and is wired into both the `TOPIC_DONE` event handler and the `stream_ask` invoke-result fallback. **Citation transform** ([`shell/src/plugins/nexus/ai/citationTransform.ts`](../../shell/src/plugins/nexus/ai/citationTransform.ts)): pure helper module split out of `ChatView.tsx` so `node:test` can exercise it without pulling React + `chat.css`. `splitTextOnCitations(text, valid)` walks `\[(\d+)\]` and emits alternating `string | { cite: number }` segments, dropping out-of-range markers as plain text so the user still sees what the model wrote. The DOM-walking wrapper `renderMarkdownWithCitations(source, validIndices)` runs `renderMarkdown` first (so code blocks / tables stay intact), then walks the resulting HTML via `DOMParser`, splitting text nodes around valid markers and substituting `<sup class="nexus-citation" data-cite="N" role="button" tabindex="0">[N]</sup>`. Marker substitution is skipped inside `<code>` and `<pre>` so code samples that contain `[1]` literals are left verbatim. The wrapper guards on `typeof DOMParser` and falls back to the unmodified rendered markdown in headless test environments. **ChatView wiring** ([`shell/src/plugins/nexus/ai/ChatView.tsx`](../../shell/src/plugins/nexus/ai/ChatView.tsx)): `AssistantBubble` swaps `MarkdownBody` for the new `CitedMarkdownBody` when `turn.citations.length > 0`; the body uses one delegated `click` listener on the container div that resolves `[data-cite]` chips back to `AiCitation` and dispatches `onCitationClick`.
