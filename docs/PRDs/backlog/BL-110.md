# BL-110: Per-frame snapshot idiom for multi-store renders

**Status:** closed — 2026-05-13

**Source**: Nexus frontend performance assessment (2026-05-11) — `experiments/nexus-frontend-assessment.html` §6
**Files**: `shell/src/stores/frameSnapshot.ts` (new); `shell/src/stores/useFrameSnapshot.ts` (new); `shell/tests/frame-snapshot.test.ts` (new, 7 tests); `shell/src/plugins/nexus/statusBar/FileStats.tsx` (migrated as the reference site)
**Related**: `experiments/zed-frontend-analysis.md` §3.1 (`EditorSnapshot`); BL-109 (the file-tree virtualization that surfaced the test-runner gap relocated here)

The Zed analogue is `EditorSnapshot`: at the start of each frame, freeze a coherent tuple of all relevant stores and render against that tuple. The React equivalent is a `useFrameSnapshot()` hook that returns a memoized tuple scoped to a `requestAnimationFrame` tick, eliminating the "two stores update in two separate microtask batches → two paints" jank without changing the architecture.

- **`FrameSnapshot` controller (React-free).** Holds the cached tuple, subscribes to N stores via the standard `Subscribable<S>` shape (`getState` + `subscribe` — Zustand's surface, also Jotai / MobX with a small adapter). On any store change, schedules one rAF flush; subsequent changes within the same frame are no-ops. On flush, recomputes per-selector values, diffs against the prior tuple via `Object.is`, and only swaps the cached reference + notifies listeners when at least one element changed. Identity stable within a frame — `current()` returns the same array reference until flush detects a diff. Scheduler is injectable via constructor so tests run synchronously without rAF / fake timers.
- **`useFrameSnapshot` hook.** Thin `useSyncExternalStore` adapter on top of the controller. Builds one `FrameSnapshot` per mount via `useMemo` (so the entries argument is captured on first render — selectors must be stable for the lifetime of the component, the standard React rule). Pairs `start()` with `subscribe()` inside the `useSyncExternalStore` subscriber so disposal happens via the same callback chain.
- **`snap()` helper for typed entries.** `snap(useEditorStore, (s) => s.tabs)` returns a typed `SnapshotEntry<EditorState, EditorTab[]>` without per-call `as const`. The hook's generic signature uses `SnapshotEntry<any, any>` at the boundary to opt out of TS variance (a tuple of heterogeneous-state entries fails assignability against `SnapshotEntry<unknown, …>` because the selector parameter is contravariant); per-element typing is recovered through the `infer V` in the result mapped type.
- **`FileStats.tsx` reference migration.** Was `useEditorStore((s) => s.tabs)` + `useEditorStore((s) => s.activeRelpath)` + `useBacklinksStore((s) => s.links.length)` + `useBacklinksStore((s) => s.loading)` — four selectors across two independent stores, both fed by async kernel events (file-watch + post-save backlinks rescan) that fire in separate microtask batches. Now one `useFrameSnapshot([…])` call with a module-level entries constant. A save-followed-by-backlinks sequence now produces one paint commit instead of two.

**Test infrastructure side-fix.** `pnpm --filter nexus-shell test` runs `node --test tests/*.test.ts` — only top-level `tests/*.test.ts`, not `src/**`. The BL-109 `flattenTree.test.ts` shipped under `src/plugins/nexus/files/` was silently uncounted. Both BL-109 and BL-110 tests relocated to `shell/tests/{flatten-tree,frame-snapshot}.test.ts` to match the convention used by `backlinks-decode.test.ts`, `bases-store.test.ts`, etc. After relocation: 1246 → 1260 tests (+14, matching 7 BL-109 + 7 BL-110). The likely-broader gap (107 src `*.test.ts` files vs 96 in `tests/`) is filed under "Suspected issues — not fully investigated" in `BACKLOG.md`.

**Tested**: `pnpm --filter nexus-shell typecheck` clean; `pnpm --filter nexus-shell lint` clean (2 expected `no-explicit-any` warnings on the variance-boundary types, scoped via `eslint-disable`); `pnpm --filter nexus-shell test` 1260/1260 pass.

**Definition of done coverage**: ✅ `useFrameSnapshot(selectors)` hook lives at `shell/src/stores/useFrameSnapshot.ts`; ✅ one hot composition site (`FileStats`) migrated as the reference; ✅ unit tests cover (a) tuple identity stability within a frame and (b) tuple invalidation when any selector value changes across frames, plus four further edge cases (no-op flush, multi-mutation coalescing, dispose cancellation, start-twice guard).
