# BL-033: AMB context chips + model switcher

**Status:** closed — 2026-04-28

**Delivered:** Closes the AI-AMBIENT-COPILOT-PLAN patterns 2 + 3 follow-on to BL-032 by giving the Cmd+I overlay both a click-to-remove chip rail and a per-overlay model switcher whose selection writes through to the global `ai.provider` / `ai.model` config keys. **Click-to-remove**: chips in [`CmdIOverlay.tsx`](../../shell/src/plugins/nexus/ai/CmdIOverlay.tsx)'s `ContextChipRail` are now `<button role="listitem">`s with a trailing "×" affordance, an aria-label and a kind-prefix glyph (📄 file, ✂ selection, ▦ block, ≡ row, ◇ node, ? query, ✎ note); clicking dispatches `useCmdIStore.removeChip(id)` (new action; the store grows a `removedChipIds: string[]` tracked alongside `chips`, reset on every `open()`). The runtime threads removal through to the prompt assembler — [`cmdIRuntime.submitCmdI`](../../shell/src/plugins/nexus/ai/cmdIRuntime.ts) builds a `Set<string>` from `removedChipIds` and passes it as the third arg to [`assemblePrompt`](../../shell/src/plugins/nexus/ai/contextContributors.ts), which gained a new `removedChipIds: ReadonlySet<string>` parameter and a new optional `chipPromptBlocks?: Record<chipId, string>` field on `ContextContribution` for chip-granular fragments. Two assembly modes coexist: **per-chip** (when `chipPromptBlocks` is set, only kept chips' fragments are emitted, in chip order) and **coarse fallback** (drop the whole surface's `promptBlock` only when ALL of its chips were removed). The editor adapter at [`editorContextAdapter.ts`](../../shell/src/plugins/nexus/ai/editorContextAdapter.ts) was reshaped to populate `chipPromptBlocks` for both the file chip and the selection chip so removing one no longer drops the other's body. **Model switcher**: a new `<ModelSwitcher>` native `<select>` sits between the chip rail and the textarea. Six presets ship today (Anthropic Opus 4.7 / Sonnet 4.6 / Haiku 4.5; OpenAI GPT-4o / GPT-4o-mini; Ollama llama3.1) — selecting one calls `useConfigStore.set('ai.provider', …)` + `set('ai.model', …)`, which fires `config:changed:ai.provider` / `config:changed:ai.model` events the AI plugin's [`index.ts`](../../shell/src/plugins/nexus/ai/index.ts) `aiKeys` listener already forwards to the kernel via `pushUserConfig` → `set_config` (handler id 1). Same keys back the Settings panel — toggles in the overlay become the new global default and persist into the chat surface and ghost completions. The control reads via `useConfigValue` so it re-renders reactively when those keys change anywhere in the app; non-preset settings (env-var fallback, custom model strings) are surfaced as a disabled `(custom)` row so the user sees what's active without us silently overwriting it. **Wire shape unchanged** — no IPC drift was needed: BL-033 reuses `set_config` for the model bind and `stream_chat` for the prompt, so `scripts/check_ipc_drift.sh` is a no-op. **Coverage**: `pnpm --filter nexus-shell test` 462 passing (was 456); new tests cover (a) `removeChip` idempotency + reset on `open()` in [`cmdIStore.test.ts`](../../shell/src/plugins/nexus/ai/cmdIStore.test.ts), (b) per-chip vs coarse removal semantics for `assemblePrompt` (3 cases) in [`contextContributors.test.ts`](../../shell/src/plugins/nexus/ai/contextContributors.test.ts), and (c) end-to-end in [`cmdIRuntime.test.ts`](../../shell/src/plugins/nexus/ai/cmdIRuntime.test.ts) where a removed `editor:selection:*` chip drops only the selection fragment from the assembled `stream_chat` payload. `pnpm --filter nexus-shell typecheck` and `pnpm --filter nexus-shell lint` clean (no new warnings).
