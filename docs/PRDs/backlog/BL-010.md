# BL-010: `nexus ai chat` interactive REPL

**Status:** closed — 2026-04-28

**Delivered:** PRD-05 §3.5.2 / §10.2's multi-turn chat REPL is now wired up as a CLI consumer of the shared `com.nexus.ai::stream_chat` engine. New [`AiCommand::Chat`](../../crates/nexus-cli/src/main.rs) variant takes `--context <FILE>` / `--model <NAME>` / `--session <ID>` / `--system <PROMPT>` flags. [`commands::ai::chat`](../../crates/nexus-cli/src/commands/ai.rs) drives a [`rustyline = "14"`](../../Cargo.toml) line editor (added as a workspace dependency), dispatching each user turn through `runtime.context.ipc_call("com.nexus.ai", "stream_chat", { mode: "chat", ...messages, session_id })` while a parallel `tokio::spawn` task drains `EventFilter::CustomPrefix("com.nexus.ai.stream_")` and prints chunks to stdout. The bus contract is exercised through one shared `stream_and_collect` pump that BL-011 also uses — same drain + same session-id correlation, so the two surfaces produce byte-identical wire events. Slash commands cover `/help` / `/clear` / `/save <FILE>` / `/model <NAME>` / `/context <FILE>` / `/system <PROMPT>` / `/quit`. `/save` writes a Markdown transcript; `/model` round-trips through `config` + `set_config` (the AI plugin's `set_config` keys on the full provider record, so a "set just the model" is a read-modify-write). Exit codes match shell convention: `0` on EOF / `/quit`, `1` on any IPC error, `130` on Ctrl-C at the prompt (mapped from `ReadlineError::Interrupted` so wrapping shells see the canonical interrupted code). Coverage: `cargo test -p nexus-cli` 45 passing including 8 new tests in [`commands::ai::tests`](../../crates/nexus-cli/src/commands/ai.rs) covering `build_chat_args` round-trip + multi-byte-safe `slice_prompt` (shared with BL-011).
