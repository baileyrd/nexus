# BL-008: Mermaid Diagram Plugin

**Status:** closed — 2026-04-28

**Delivered:** New fenced-code-renderer registry — [`shell/src/plugins/nexus/editor/cm/fencedCodeRegistry.ts`](../../shell/src/plugins/nexus/editor/cm/fencedCodeRegistry.ts) — owns a singleton `Map<language, FencedRenderer>` with first-match-wins duplicate guard, a per-(language, source) LRU cache (cap 64), `generation()` counter, and `onChange()` listeners that the live-preview StateField subscribes to via a `ViewPlugin` so a renderer registered after mount triggers a recompute. Public API: `api.editor.registerFencedCodeRenderer(language, renderer)` exposed via [`@nexus/extension-api` `EditorAPI`](../../packages/nexus-extension-api/src/index.ts) and wired through [`shell/src/host/PluginAPI.ts`](../../shell/src/host/PluginAPI.ts) — the disposer is auto-tracked by `PluginRegistry.trackSubscription` so plugin deactivate sweeps the registration. Live-preview integration: new `FencedCodeWidget` adjacent to `TableWidget` in [`livePreviewDecorations.ts`](../../shell/src/plugins/nexus/editor/cm/livePreviewDecorations.ts); off-cursor blocks with a registered language emit a block-replace widget that calls `renderCached` (sync hit → swap inline; miss → `<pre><code>` placeholder + async swap on resolve). On-cursor falls through to the legacy fence-line replace so source stays editable. Preview-mode integration: `marked.use({ renderer: { code } })` in [`markdownRender.ts`](../../shell/src/plugins/nexus/editor/markdownRender.ts) emits `<div class="nexus-fenced-pending">` placeholders carrying the language + base64 source; the new `hydrateFencedCode(root)` helper walks them after React mounts the sanitized HTML and replaces each with the rendered widget. Mermaid plugin lives under [`shell/src/plugins/community/mermaid/`](../../shell/src/plugins/community/mermaid/) — `index.ts` registers the renderer and lazy-imports the ~150 KB mermaid library only on first render via Vite's chunked dynamic import. Default-off in `shell/src/plugins/catalog.ts` (DEFAULT_OFF_PLUGINS) so users opt in via Settings → Plugins. Coverage: 14 new unit tests in `fencedCodeRegistry.test.ts` covering register / dispose / duplicate-warn / case-insensitivity / generation / onChange / sync + async + error caching / re-register invalidation; 4 new live-preview decoration tests for FencedCodeWidget emission, on-cursor fallback, unregistered-language pass-through, and widget equality; 3 mermaid-plugin lifecycle tests. Deviation from the plan: the spec called for a community-plugin loaded via the Blob-URL community-plugin loader, but that path can't resolve `import("mermaid")` without a community-plugin bundler. The directory layout matches the spec; registration flows through the catalog as a stepping-stone — `plugin.json` is `enabled: false` so the community loader skips it. Once a community-plugin bundler lands, the catalog wiring will be deleted and the plugin will load through the standard discovery path.
