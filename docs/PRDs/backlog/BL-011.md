# BL-011: `nexus ai complete` CLI

**Status:** closed — 2026-04-28

**Delivered:** PRD-05 §3.5.3 / §10.3's headless completion CLI ships as the twin of BL-034 — same engine call shape (`mode = "complete"`, `tools = "none"`, `trim = true`, `max_tokens = 64`, `stop = ["\n\n"]`), different rendering boundary. New [`AiCommand::Complete { file, line, col, context_lines }`](../../crates/nexus-cli/src/main.rs) variant + [`commands::ai::complete`](../../crates/nexus-cli/src/commands/ai.rs) reads the file via `storage_ipc::read_file`, slices the prefix at `(--line, --col)` (1-based, char-boundary-safe so multi-byte codepoints aren't split), optionally clamps to the last `--context N` lines, then issues a single `stream_chat` call through the same `stream_and_collect` pump BL-010 uses. Engine post-processing (prompt-echo strip + natural-break clip + stop-sequence truncation) runs server-side per the shared engine contract — the CLI just prints the resulting `text`. Coverage: 6 of the 8 new `commands::ai::tests` cases target `slice_prompt` directly (whole-file default, line-end clip, mid-line col clip, context-window cap, multi-byte safety, line-overshoot clamp); the engine call shape is exercised end-to-end by the BL-034 ghost path.
