# BL-086: Auto-commit scheduler activation

**Status:** closed — 2026-05-06

**Source**: Git Integration Assessment (2026-05-06) — gap #2
**Files**: `crates/nexus-git/src/core_plugin.rs`, `crates/nexus-git/Cargo.toml`, `crates/nexus-cli/src/`

`GitCorePlugin::on_start` now reads `.forge/app.toml` for `[git] auto_commit = true` (default false, opt-in). When enabled it spawns a `nexus-git-auto-commit` thread that wakes every 30 s, drains `com.nexus.storage.file_modified` events from the EventBus to track the last-edited timestamp, and calls `AutoCommitter::check_and_commit` once the forge has been idle for `auto_commit_interval_secs` (default 1800 s). Successful auto-commits publish `com.nexus.activity.appended { origin: "git:auto-commit", ... }`. The thread is joined cleanly in `on_stop`. CLI: `nexus git auto-commit --enable` and `--disable` write the TOML key to `.forge/app.toml` preserving all other settings; restart required to apply.
