# BL-017: AI PII / Secret Egress Filter

**Status:** closed — 2026-04-28

**Delivered:** PRD-12 §15.1 lands as a small library-only addition to `nexus-ai`. New [`crates/nexus-ai/src/privacy.rs`](../../crates/nexus-ai/src/privacy.rs) defines `PrivacyPolicy` (`Off` default, `Redact`, `Strict` — `#[derive(Default, Serialize, Deserialize)]`, `#[serde(rename_all = "lowercase")]`), `Redaction { pattern_id: &'static str, start: usize, end: usize }` records (byte offsets into the *original* input — useful for diagnostics), and a `Redactor` that compiles a default set of six high-confidence secret patterns under `regex-lite` 0.1 (already a workspace dep — no new external crate; the full-fat `regex` was deliberately avoided). Pattern set: `aws-access-key` (`AKIA[0-9A-Z]{16}` → `[REDACTED:aws-access-key]`), `aws-secret` (`(?i)aws(.{0,20})?(secret|access)?(.{0,20})?[=:\s]+[A-Za-z0-9/+=]{40}` → `[REDACTED:aws-secret]` — verified `regex-lite` accepts the leading `(?i)` flag and `[\s\S]` character class on its own; no syntax adjustments were required), `api-token` (`sk_(live|test)_[A-Za-z0-9]{24,}` → `[REDACTED:api-token]` for Stripe-style keys), `github-pat-classic` (`ghp_[A-Za-z0-9]{36}` → `[REDACTED:github-token]`), `github-pat-fine` (`github_pat_[A-Za-z0-9_]{82}` → `[REDACTED:github-token]`), and `private-key` (PEM `-----BEGIN [A-Z ]*PRIVATE KEY-----[\s\S]*?-----END [A-Z ]*PRIVATE KEY-----` → `[REDACTED:private-key]`). API surface: `Redactor::with_default_patterns()` / `with_strict()` / `for_policy(PrivacyPolicy) -> Option<Self>` (returns `None` for `Off` — the convention callers thread through the prompt builder), `redact(&self, &str) -> (String, Vec<Redaction>)` and the in-place variant `redact_in_place(&self, &mut String) -> Vec<Redaction>`, plus `is_strict()` returning the configured intent so a future error-on-match wiring can branch without an enum-shape break. Wire point is intentionally narrow: [`build_rag_prompt_budgeted`](../../crates/nexus-ai/src/rag.rs) gained a fourth `redactor: Option<&Redactor>` parameter that runs each accepted chunk's text through `redact_in_place` before it's charged against the token budget and stitched into the prompt body. `stream_chat`, the Anthropic / OpenAI / Ollama provider HTTP request bodies, and the `AiCorePlugin` dispatcher are deliberately untouched — RAG injects retrieved content the user didn't paste this turn, which is the right boundary; silently mutating the user's typed prompt would be surprising and the user already chose to send what they pasted. The legacy `build_rag_prompt(sources)` wrapper threads `None` through so its byte-output stays identical to the BL-018 contract (verified by `budgeted_prompt_passes_through_when_redactor_is_none`). `AiConfig` gained `pub privacy: PrivacyPolicy` defaulting to `Off` — opt in by setting it to `Redact` (or `Strict`) and constructing a `Redactor` via `Redactor::for_policy(...)`. `PrivacyPolicy::Strict` is shipped today as a no-op alias of `Redact`: it constructs the same redactor and replaces the same matches with the same placeholders, but `Redactor::is_strict()` returns `true` so when a downstream caller wants the error-on-match path (e.g. refusing to send the prompt at all when a secret is detected), the wiring is a one-line check on the redactor without re-plumbing the config enum. Coverage: 7 new unit tests in `privacy.rs` (`redactor_replaces_aws_access_key`, `redactor_replaces_private_key_block_multiline`, `redactor_replaces_multiple_distinct_secrets_in_one_pass` covering Stripe key + GitHub PAT in one input, `redactor_no_match_returns_input_unchanged` proving clean text round-trips with an empty event vec, `for_policy_returns_none_for_off`, `is_strict_reflects_constructor`, `privacy_policy_default_is_off`) plus 2 new tests in `rag.rs::tests` (`budgeted_prompt_redacts_chunk_text_when_redactor_supplied`, `budgeted_prompt_passes_through_when_redactor_is_none` — the BL-018 byte-identity guard). Total `cargo test -p nexus-ai`: 31 passing (was 22). Out of scope per the BL-017 plan: no IPC handler (the existing `AiCorePlugin` dispatcher is untouched), no `nexus-security` integration, no error path on `Strict`, no plumbing through `stream_ask` — those ship when a downstream caller needs them.
