## Risk classification

`crates/nexus-security/src/risk.rs::risk_level(Capability) -> RiskLevel`.

- **High** (orange/red in install-time UI): `fs.{read,write}.external`, `net.http`, `process.spawn`, `ipc.call`, `ai.config.write`, `audio.record`, `protocol.host.contribute`.
- **Medium**: `fs.write`, `net.http.localhost`, `db.{query,write}`, `events.publish`, `ai.chat`, `ai.activity.write`, `ai.tools.{write,mcp}`, `ai.runtime.{submit,control}`.
- **Low**: `fs.read`, `kv.{read,write}`, `ui.notify`, `ai.{index, session.read, session.write}`, `ai.runtime.observe`, `audio.synthesize`, `notifications.inbox.{read,write}`.

## How a plugin gets a capability

```toml
# plugin.toml
[capabilities]
required = ["fs.read", "kv.read"]
optional = ["net.http"]
```

At install:
1. The shell reads `manifest.capabilities` and shows the user the union, with HIGH-risk verbs called out (`shell/src-tauri/src/lib.rs::get_plugin_granted_capabilities`).
2. The user grants a subset. The grant is persisted to `<plugin_dir>/granted_caps.json` (chacha20poly1305-sealed, key in OS keyring per BL-101).
3. At runtime, `nexus-kernel::IpcDispatcher::dispatch` checks each call against the granted set. Missing cap → `CapabilityDenied`.

## Audit trail

Every cap-gated call is logged through `nexus-security::audit` and persisted to `<forge>/.forge/.kernel/audit.db` (90-day retention by default; queryable via `com.nexus.security::query_audit_log`).

## Open questions (AUDIT flags)

`cap_matrix.toml` carries `# AUDIT:` comments for handlers whose current `unrestricted` classification preserves pre-BL-138 behaviour but is a candidate for cap elevation. The full list is at [`reference/audit-flags.md`](reference/audit-flags.md). Highlights:
- `com.nexus.security::{set_secret, delete_secret, clear_audit_log}` — currently `ipc.call` only; a hostile plugin could overwrite secrets or erase audit history.
- `com.nexus.ai::resolve_credentials` — returns keyring material; candidate for an in-tree-only marker.
- `com.nexus.terminal::{send_input, send_raw_input, run_saved, adhoc_promote, repl_eval}` — writes into an already-spawned PTY; candidate for `process.spawn` elevation.
- `com.nexus.git::{push, push_tags}` — outbound network reach; candidate for `net.http`.
- `com.nexus.linkpreview::fetch` — outbound HTTP; candidate for `net.http`.
- `com.nexus.workflow::{run, run_digest}` — laundering surface: each step is gated by the target plugin's caps, but a workflow can chain calls. Tracked under issue #77 and BL-134 Phase 3.
- `com.nexus.collab::start_relay` — binds 0.0.0.0; candidate for a future `network.bind` cap.
