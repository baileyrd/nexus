Target Architecture

Four-tier crate DAG. Frontends attach to services only through context.ipc_call(plugin_id, cmd, args) backed by the kernel event bus. The new plugin-first shell is the single desktop target; the legacy app/ + crates/nexus-app tree was deleted on 2026-04-24 (Phase 4 WI-37).

IPC Flow (Desktop Shell → Service Plugin)

Every subsystem call — read a file, ask the AI, run a workflow, toggle a theme — flows through this four-hop contract. Tauri commands are thin adapters; no business logic lives in the host.

Shell (webview)

React plugin calls the bridge

api.kernel.invoke(
 'com.nexus.storage',
 'search', args)
→

Tauri host

Bridge command serializes + dispatches

#[tauri::command]
kernel_invoke(...)
→

Kernel + service

Service plugin handles IPC and returns JSON

ctx.ipc_call(...)
↓
CorePlugin::handle_ipc

Bus events flow the other way via topic subscriptions: com.nexus.storage.file_created, com.nexus.theme.changed, com.nexus.ai.stream_chunk, com.nexus.agent.step_done, com.nexus.editor.changed.<relpath>. The shell subscribes via api.kernel.on(topic, handler).

Phased Roadmap

Estimated ~16–24 engineer-weeks to v1. Phase 2 is the largest item and captures most calendar time.

Phase 0~1 week
Decision & Freeze
  • Accept ADR 0011 — shell/ is the single desktop target
  • Tag legacy at v0.1.0-legacy-shell, add DEPRECATED banner
  • Freeze new capability work on the legacy app/ + crates/nexus-app tree (since deleted 2026-04-24)
  • Derive the ~95-handler parity checklist from the then-legacy crates/nexus-app per-module surface (now ported to service-crate core_plugin.rs IPC handlers)
Accept: checklist published, freeze in effect.
Phase 12–3 weeks
Bridge & Leaf Foundation
  • Complete Leaf/ViewRegistry impl (leaf-migration-plan Phases 0–4)
  • Finalize shell-kernel bridge Phase 4 polish
  • Wire @nexus/extension-api (already scaffolded in packages/, unused); regenerate types via ts-rs
  • Add shell-side dep-invariants guardrail (no raw Tauri API from plugins)
Accept: empty shell + one hello-world plugin + round-trip kernel event.
Phase 26–10 weeks
Feature Parity Migration
  • Editor transaction wiring (Phases 0–8), then file tree / tab strip / markdown view
  • Notion blocks → Canvas → Bases → Outline/Backlinks/Search → Graph
  • AI chat + Agent planner with streaming event subscriptions
  • Terminal with xterm.js and com.nexus.terminal::* IPC; convert output to streaming events
  • Workflow · Skills · MCP management panels, settings, keybindings editor, command palette
  • Status bar · activity bar · titlebar (already plugins in shell/src/plugins/core/)
Accept: ~95-handler parity checklist passes; side-by-side workflow test passes.
Phase 33–4 weeks
Security Hardening
  • JS plugin iframe sandbox + CSP re-enable (UI F-8.1.1)
  • Install-time capability prompt (MK F-5.1.1) with risk classification
  • TOCTOU fixes in host write paths (MK F-5.3.1, F-5.3.2)
  • api_version range check at load time (MK F-9.2.1, UI F-9.1.1)
  • Per-plugin crash quarantine (stretch)
Accept: all 🔴 audit findings closed; security review passes.
Phase 4~2 weeks
Frontend Unification Polish
  • Normalize command taxonomy across CLI / TUI / MCP / desktop
  • Retire crates/nexus-app; delete app/ — shipped 2026-04-24 under WI-37
  • Single nexus binary: nexus desktop, nexus tui, nexus mcp, headless subcmds
  • nexus plugin scaffold emits manifest + TS + WASM stubs together
  • MCP parity: every MCP tool has a shell command and CLI subcommand
Accept: one binary, four modes, zero drift.
Phase 53–4 weeks
v1 Polish & Ship
  • Tauri updater + signing + release channel
  • Opt-in crash reporting (never upload note content)
  • Bundled curated core plugin set
  • Minimal static marketplace index + install flow
  • Docs pass · beta · v1.0.0
Accept: GA tag, public release.
